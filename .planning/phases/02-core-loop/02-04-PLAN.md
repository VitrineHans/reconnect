  <action>
    BEFORE writing any code, execute the frontend-design skill's design thinking process:

    **Context for the skill:**
    - App: Reconnect — intimate 1:1 friendship app; daily ephemeral video Q&A
    - Audience: Young adults (20s–30s) who want deeper connection with close friends
    - Tone to commit to: warm but bold; feels personal and slightly electric; the reveal moment is
      the peak — needs to feel like opening a gift; NOT corporate, NOT playful-startup, NOT dating-app sleek
    - Core emotion: curiosity, warmth, anticipation → delight
    - One thing users will remember: the Reveal Ready card should feel unmistakably different
      from the waiting states — it should pulse with quiet excitement

    **Design decisions to make (run through the skill's framework):**
    1. Colour palette: Choose a primary background (not white, not black), one dominant accent for
       reveal/active states, one secondary accent, neutral greys for waiting states.
       Consider: warm off-white or deep warm dark background? Charcoal backgrounds make reveals glow.
    2. Typography: Pick two fonts available via @expo-google-fonts or system fonts.
       Display/heading font: something with character (NOT Inter, NOT Space Grotesk).
       Body font: highly legible at small sizes.
       Sizes: establish a 6-step scale (xs, sm, md, lg, xl, 2xl).
    3. Spacing: 4pt base grid; define scale: 4, 8, 12, 16, 24, 32, 48, 64
    4. Motion concept: reveal card "open" feeling (Phase 3 can add animation; tokens provide the
       colour treatment that makes it feel distinct without motion)
    5. Border radii: rounded cards (16px), pill badges (999px)
    6. Shadows: card elevation for reveal-ready state

    After the design thinking step, create mobile/theme/tokens.ts as a TypeScript const object:

    ```typescript
    // mobile/theme/tokens.ts
    export const colors = {
      background: string,    // primary app background
      surface: string,       // card surface
      surfaceAlt: string,    // slightly different surface for waiting states
      accent: string,        // reveal-ready, active CTAs
      accentMuted: string,   // your-turn state
      textPrimary: string,
      textSecondary: string,
      textMuted: string,
      streak: string,        // flame badge background
      border: string,
      error: string,
    } as const;

    export const typography = {
      fontFamily: {
        display: string,     // heading/display font name
        body: string,        // body font name
      },
      fontSize: {
        xs: 11,
        sm: 13,
        md: 16,
        lg: 20,
        xl: 28,
        xxl: 38,
      },
      fontWeight: {
        regular: '400' as const,
        medium: '500' as const,
        semibold: '600' as const,
        bold: '700' as const,
        black: '900' as const,
      },
      lineHeight: {
        tight: 1.2,
        normal: 1.5,
        relaxed: 1.7,
      },
    } as const;

    export const spacing = {
      xs: 4, sm: 8, md: 12, lg: 16, xl: 24, xxl: 32, xxxl: 48,
    } as const;

    export const radii = {
      sm: 8, md: 12, lg: 16, xl: 24, pill: 999,
    } as const;

    export const shadows = {
      card: {
        shadowColor: string,
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.10,
        shadowRadius: 8,
        elevation: 3,
      },
      cardReveal: {
        shadowColor: string,   // accent-coloured shadow for reveal-ready cards
        shadowOffset: { width: 0, height: 4 },
        shadowOpacity: 0.25,
        shadowRadius: 16,
        elevation: 8,
      },
    } as const;
    ```

    Commit to the design direction chosen by the skill. Fill in all string values with
    deliberate hex codes. The accent colour for reveal-ready state should be genuinely
    striking — not a generic blue or purple. Consider warm amber, electric coral, deep teal,
    or midnight indigo with a luminous quality.

    NOTE: If @expo-google-fonts fonts are chosen, add them to the installation list in the
    action section below. If using system fonts only, no installation needed.
  </action>
