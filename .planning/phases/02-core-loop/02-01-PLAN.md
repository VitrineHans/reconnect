## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase DB | question_ratings INSERT/UPDATE; question_responses SELECT |
| Supabase DB → pg_cron | rotate_daily_questions() runs as security definer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | question_responses SELECT RLS | mitigate | New SELECT policy gates partner response visibility on count(*) = 2; replaces broad existing policy |
| T-02-01-02 | Tampering | question_ratings upsert | mitigate | RLS WITH CHECK (auth.uid() = user_id) prevents rating on behalf of other users |
| T-02-01-03 | Elevation of Privilege | rotate_daily_questions() SECURITY DEFINER | accept | Function has no user-facing input; only reads/writes own tables; pg_cron trigger only |
| T-02-01-04 | Spoofing | friendships query (no ownership check) | mitigate | RLS on friendships table (existing from init migration) enforces user_a/user_b membership |
</threat_model>
