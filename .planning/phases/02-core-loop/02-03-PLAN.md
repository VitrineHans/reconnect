<!-- Realtime subscription pattern (RESEARCH.md Pattern 7) -->
```typescript
// CRITICAL: all .on() calls BEFORE .subscribe()
const channel = supabase
  .channel(`responses:${friendshipId}:${questionId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'question_responses',
    filter: `friendship_id=eq.${friendshipId}`,
  }, async (_payload) => {
    const { data } = await supabase
      .from('question_responses')
      .select('user_id')
      .eq('friendship_id', friendshipId)
      .eq('question_id', questionId);
    if (data?.length === 2) onBothSubmitted();
  })
  .subscribe();
// cleanup: return () => { supabase.removeChannel(channel); }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase Realtime | postgres_changes subscription; payload not trusted for state |
| client → Supabase Storage | createSignedUrl for read; remove() for deletion |
| client → supabase/functions/notify-reveal | Invoked from DB trigger (Phase 3); service role key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | reveal.tsx fetching partner video | mitigate | RLS from 02-01 enforces count=2 gate; createSignedUrl only generates URL when RLS allows SELECT |
| T-02-03-02 | Information Disclosure | Signed URL in memory | accept | 5-min expiry; URL not persisted to DB (RESEARCH Pitfall 5); ephemeral in component state |
| T-02-03-03 | Tampering | VideoPlayer deletion | mitigate | Storage DELETE RLS verifies path segment [2] = auth.uid(); user can only delete own video |
| T-02-03-04 | Repudiation | watched_at not set if video not fully watched | accept | MVP: set watched_at on playToEnd only; partial watches do not delete video |
| T-02-03-05 | Denial of Service | Realtime subscription memory leak | mitigate | useRevealSubscription cleanup returns supabase.removeChannel(channel) in useEffect teardown (RESEARCH Pitfall 4) |
| T-02-03-06 | Elevation of Privilege | notify-reveal Edge Function | mitigate | Uses SUPABASE_SERVICE_ROLE_KEY from environment; not exposed to client; Phase 3 wires DB trigger |
</threat_model>
